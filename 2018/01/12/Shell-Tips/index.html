git clone时出错

For a permanent fix, choose one of

  • On the older host, set the LANG environment variable in your shell’s initialization file.
  • Modify your environment on the client side, e.g., rather than ssh hunter2, use the command LANG=C ssh hunter2.
  • If you have admin rights, stop ssh from sending the environment variables by commenting out the SendEnv LANG LC_* line in >the local /etc/ssh/ssh_config file. (Thanks to this answer. See Bug 1285 for OpenSSH for more.)

系统无法识别硬盘时参考方案

  1. 重启进入BIOS，查看硬盘是否可以识别，如果可以，继续往下。如果BIOS识别不到，无解。
  2. 使用开机引导U盘引导，进入U盘linux或临时系统
  3. 使用lsblk,df -h等命令查看无法识别的硬盘能否看到
  4. 如果可以看到，则可以外挂U盘，将此盘的内容拷贝出来。拷贝命令可以参考上一条
  5. 可以用拷贝后的硬盘替换无法识别的硬盘，或尝试重装系统，看能否识别原硬盘
